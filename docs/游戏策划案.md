# 小游戏策划案

## 1. 游戏定位

本项目是一款结合自走棋养成和肉鸽成长的微信小游戏。玩家通过刷新、购买、合成棋子来组建阵容，抵御一波波怪物进攻，并在局内不断升级商店等级、提高阵容强度。

核心体验关键词：

- 自走棋：随机商店、购买棋子、三合一升阶、自动寻敌战斗。
- 肉鸽：每局从低阶棋子开始，通过随机选择和资源分配形成不同成长路线。
- 轻量化：单局节奏快，操作集中在刷新、购买、升级和阵容摆放。

## 2. 核心玩法循环

1. 开局获得 1 个【凡人】棋子和少量初始金币。
2. 怪物在开局 5 秒后开始分波出现。
3. 玩家击杀怪物获得金币。
4. 玩家消耗金币刷新棋子商店，购买更强棋子。
5. 棋盘上 3 个相同类型的棋子自动合成为下一等级棋子，并恢复血量。
6. 玩家消耗金币提升等级，提高高阶棋子出现概率。
7. 随着关卡时间推进，怪物数量、等级和强度逐步提升。
8. 玩家尽可能坚持更长时间，形成更高阶阵容。

## 3. 当前界面与节点规划

战斗界面以 `GamePanel` 为核心，主要节点如下：

| 节点/组件 | 用途 |
| --- | --- |
| `m_CharacterRoot` | 棋盘根节点，承载玩家棋子和敌人棋子 |
| `m_CharacterItem` | 棋子预制模板，用于克隆玩家棋子或敌人棋子 |
| `m_CharacterItem/CharacterType` | `Label`，显示棋盘上该棋子的类型名称 |
| `m_CharacterItem/Hp` | `ProgressBar`，显示棋盘上该棋子的当前血量比例 |
| `m_CharacterItem/ItemBtn` | `Button`，作为棋子购买按钮，点击时购买该棋子 |
| `m_ItemSelectLayout` | 棋子随机商店列表容器 |
| `m_Item` | 商店中的单个棋子商品模板 |
| `m_Item/CharacterType` | `Label`，显示棋子类型名称 |
| `m_Item/Coin` | `Label`，显示购买该棋子的金币消耗 |
| `m_RefreshBtn` | 刷新商店按钮 |
| `m_LvUpBtn` | 玩家等级提升按钮 |
| `m_TitleText` | 顶部提示文本，用于显示下一波怪物倒计时、当前波次和关键战斗提示 |
| `m_LvText` | 显示玩家当前等级，初始为 `Lv.1` |

## 4. 棋子体系

### 4.1 玩家棋子路线

玩家棋子共有 3 条成长路线，均从【凡人】开始。不同路线的成长速度、资源需求和最终强度不同。

| 路线 | 定位 | 成长链路 |
| --- | --- | --- |
| 修仙系 | 成长慢，消耗资源极高，后期战力惊人 | 【凡人】->【练气】->【筑基】->【结丹】->【元婴】->【化神】->【炼虚】->【合体】->【渡劫】->【大乘】->【天仙】->【真仙】->【玄仙】->【金仙】->【太乙】->【大罗】->【混元】 |
| 超凡系 | 成长快，资源需求适中，成型早，战力中等 | 【凡人】->【觉醒】->【精英】->【大师】->【史诗】->【传奇】 |
| 武道系 | 成长速度中等，资源需求适中，战力稳定 | 【凡人】->【武者】->【武师】->【武宗】->【武王】->【武圣】->【武尊】->【武皇】->【武帝】->【武神】 |

### 4.2 敌人棋子路线

敌人棋子随波次和关卡时间逐步升级。

【妖兽】->【小妖】->【大妖】->【妖将】->【妖王】->【妖皇】->【妖帝】->【妖神】->【妖仙】

### 4.3 基础属性规则

棋子级别越高，血量和攻击力越高。建议首版先使用简单线性或倍率公式，后续再做精细数值表。

推荐基础规则：

- 血量：`基础血量 * 等级倍率 * 路线系数`
- 攻击：`基础攻击 * 等级倍率 * 路线系数`
- 攻击间隔：默认 1 秒，可由不同路线微调。
- 移动速度：按格移动，每格大小为 `80 * 80`。

路线系数建议：

| 路线 | 血量系数 | 攻击系数 | 说明 |
| --- | --- | --- | --- |
| 修仙系 | 中 | 高 | 后期主输出，前期获取难度高 |
| 超凡系 | 中 | 中 | 成型快，适合前中期过渡 |
| 武道系 | 高 | 中 | 生存强，适合作为前排 |
| 敌人 | 随波次提升 | 随波次提升 | 用于制造持续压力 |

## 5. 经济系统

### 5.1 金币来源

- 击杀怪物获得金币。
- 怪物等级越高，掉落金币越多。
- 后续可扩展为波次奖励、连胜奖励、任务奖励。

### 5.2 金币消耗

- 刷新商店：点击 `m_RefreshBtn` 消耗金币，随机出现 5 张棋子。
- 购买棋子：点击棋子上的 `m_CharacterItem/ItemBtn`，消耗对应金币。
- 升级玩家等级：点击 `m_LvUpBtn` 消耗金币，提高玩家等级。

### 5.3 推荐首版数值

| 项目 | 建议值 |
| --- | --- |
| 初始金币 | 10 |
| 刷新消耗 | 2 |
| 初始等级 | Lv.1 |
| 商店单次刷新数量 | 5 |
| 棋子购买价格 | 根据路线和阶级递增 |

棋子价格建议：

- 超凡系：价格低，便于快速成型。
- 武道系：价格中等，成长稳定。
- 修仙系：价格高，后期强度高。

## 6. 玩家等级与商店概率

玩家等级影响商店中高阶棋子的出现概率。等级越高，高阶棋子和稀有路线出现概率越高。

推荐首版等级设计：

| 玩家等级 | 升级消耗 | 商店倾向 |
| --- | --- | --- |
| Lv.1 | - | 主要出现【凡人】和低阶超凡/武道棋子 |
| Lv.2 | 6 | 少量出现低阶修仙棋子 |
| Lv.3 | 10 | 中阶超凡/武道概率提升 |
| Lv.4 | 16 | 修仙系中阶开始出现 |
| Lv.5 | 24 | 高阶棋子概率提升 |
| Lv.6 | 36 | 稀有高阶棋子成为主要追求 |

商店刷新规则：

- 每次刷新生成 5 个商品，可以出现重复类型。
- 生成结果根据玩家等级读取权重。
- 已购买的商品节点隐藏，下一次刷新时重新显示并更新内容。
- 金币不足时点击商品，给出提示，不扣除金币。

## 7. 棋子购买与上场规则

点击商店棋子上的 `m_CharacterItem/ItemBtn` 时：

1. 读取商品对应棋子类型和价格。
2. 判断玩家当前金币是否足够。
3. 金币不足：弹出提示，例如“金币不足”。
4. 金币足够：扣除金币，隐藏该商品节点。
5. 从商品节点位置克隆一个 `m_CharacterItem`。
6. 将新棋子挂到 `m_CharacterRoot` 下。
7. 在棋盘底部查找一个未占用格子放置棋子，避免重叠。
8. 棋子进入自动战斗逻辑。

棋盘摆放建议：

- 棋盘按 `80 * 80` 的格子管理。
- 玩家棋子优先放在底部空格。
- 敌人棋子优先从顶部或边缘生成。
- 每个格子只允许存在一个棋子。

## 8. 战斗系统

### 8.1 寻敌规则

棋子进入战斗后自动寻找最近敌人：

- 玩家棋子寻找最近敌方棋子。
- 敌人棋子寻找最近玩家棋子。
- 如果没有目标，则原地等待或向对方区域推进。

### 8.2 移动规则

- 棋子以格子为单位移动，每格大小为 `80 * 80`。
- 每次向目标方向移动一格。
- 移动时需要避开已被占用的格子。
- 当棋子进入目标周边一格范围时停止移动，进入攻击状态。

### 8.3 攻击规则

攻击表现为一次快速位移：

1. 棋子从当前位置快速移动到目标所在格子。
2. 命中时根据攻击力扣除目标血量，并同步刷新目标 `m_CharacterItem/Hp` 的进度值。
3. 棋子快速返回原格子。
4. 等待攻击间隔后再次判断目标状态。

如果目标死亡：

- 目标节点从 `m_CharacterRoot` 移除。
- 如果死亡的是敌人，玩家获得金币。
- 如果死亡的是玩家棋子，只移除棋子，不返还金币。
- 攻击者重新寻找新的目标。

## 9. 合成系统

当 `m_CharacterRoot` 上存在 3 个相同类型的玩家棋子时，自动合成为下一等级棋子。

合成规则：

- 只合成玩家棋子，敌人不参与合成。
- 3 个同名棋子合成 1 个下一阶棋子。
- 合成后的棋子恢复满血。
- 合成位置优先使用 3 个棋子中最靠近棋盘底部的格子。
- 被消耗的 2 个棋子节点移除。
- 如果该棋子已经是该路线最高阶，则不再合成。

示例：

- 3 个【凡人】可以根据设计选择合成为指定路线的一阶棋子，或暂时合成为【练气】作为首版规则。
- 3 个【武者】合成为 1 个【武师】。
- 3 个【觉醒】合成为 1 个【精英】。

首版建议：先让同路线棋子按固定路线合成；【凡人】作为通用起点，购买时即确定所属路线，避免合成分支过于复杂。

## 10. 怪物波次系统

关卡开始 5 秒后开始出怪，之后按波次持续刷新。

基础规则：

- 每波怪物数量：3 到 20 只。
- 每波间隔：20 到 60 秒。
- 波次越高，怪物数量越多。
- 关卡时间越长，怪物等级越高。
- 玩家等级越高，系统可适当提高怪物强度，避免玩家滚雪球过快。

推荐首版成长公式：

- 怪物数量：`min(3 + 当前波次 * 2, 20)`
- 怪物等级：根据 `关卡时间 / 60秒` 和 `当前波次` 共同决定。
- 波次间隔：前期 30 秒，后期逐渐缩短到 20 秒。

波次提示规则：

- `m_TitleText` 用于显示下一波怪物抵达倒计时，例如“第 3 波怪物将在 12 秒后抵达”。
- 怪物即将刷新前 3 秒可显示强调提示，例如“怪物即将抵达！”。
- 怪物刷新后显示当前波次信息，例如“第 3 波怪物来袭”。
- 非波次关键时刻也可复用该文本显示提示，例如“金币不足”“棋子已合成”“敌人数量过多”。

## 11. 失败与结算

首版可采用简单失败条件：

- 当玩家棋盘上没有任何玩家棋子，并且金币不足以购买棋子时，游戏失败。
- 或者当敌人突破指定防线/存活敌人数量超过上限时，游戏失败。

结算内容建议：

- 存活时间。
- 击杀怪物数量。
- 最高玩家等级。
- 最高棋子阶级。
- 获得金币总量。

## 12. 首版 MVP 开发范围

建议先完成最小可玩闭环，再继续扩展肉鸽内容。

### 12.1 必做功能

- 玩家金币、等级显示。
- 商店刷新，生成 5 个棋子商品。
- 商品购买、扣金币、金币不足提示。
- 玩家棋子生成到棋盘空格。
- 敌人按波次生成。
- 玩家棋子和敌人自动寻敌、移动、攻击、死亡。
- 击杀敌人获得金币。
- 3 合 1 合成并恢复血量。

### 12.2 可延后功能

- 不同路线专属技能。
- 肉鸽随机事件。
- 局外成长。
- 图鉴和成就。
- 复杂寻路和阵型策略。
- 更完整的数值表和难度曲线。

## 13. 后续扩展方向

- 加入棋子技能，例如范围攻击、治疗、护盾、召唤。
- 加入路线羁绊，例如 3 个武道系提高全体血量。
- 加入肉鸽奖励，例如每 3 波选择一个被动强化。
- 加入 Boss 波次，周期性检验阵容强度。
- 加入局外养成，提升开局金币、刷新概率或指定路线概率。
