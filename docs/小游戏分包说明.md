# 小游戏分包说明

本文档用于说明 Cocos Creator 小游戏项目如何做分包，以及分包和远程资源的区别。

## 1. 分包是什么

小游戏分包是微信小游戏、抖音小游戏等平台提供的包体拆分能力。

项目上传审核时，主包和分包会作为同一个小游戏版本一起提交到平台后台。玩家进入游戏时，客户端会先下载主包；当游戏运行到需要某个分包的功能时，再由平台从自己的 CDN 下载对应分包。

因此，小游戏分包一般不需要开发者自己放到业务服务器上，也不需要自己写 HTTP 下载逻辑。

## 2. 分包和远程资源的区别

| 类型 | 是否需要自己放服务器 | 下载方 | 常见用途 |
| --- | --- | --- | --- |
| 小游戏分包 | 不需要 | 微信/抖音等平台 CDN | 降低首包体积，延迟加载游戏功能 |
| 远程资源 | 需要 | 自己的 CDN 或资源服务器 | 热更新、活动资源、可独立更新的大资源 |

如果只是为了减少小游戏首包大小，优先使用平台分包。

如果希望资源不跟随小游戏版本审核，而是由服务器动态下发，才考虑远程资源。

## 3. 推荐拆分原则

主包只放玩家进入游戏后立刻需要的内容：

- 启动场景。
- 登录界面。
- 首页界面。
- 基础 UI 框架。
- 平台初始化逻辑。
- 必须立即显示的字体、图标、加载页资源。

分包放后续功能才会用到的内容：

- 战斗场景。
- 关卡资源。
- 排行榜界面资源。
- 皮肤、角色、装饰资源。
- 大音频、大特效、大图集。
- 活动玩法资源。

不要把首次进入游戏就必须用的内容放进分包，否则玩家可能会在启动阶段等待分包下载，影响首屏体验。

## 4. Cocos Creator 中如何配置分包

Cocos Creator 通常通过 Asset Bundle 来配置小游戏分包。

示例目录结构：

```text
assets/
  resources/              # 主包常用资源
  scripts/                # 主包脚本
  subpackages/
    game/                 # 战斗或关卡分包
    rank/                 # 排行榜相关分包
    skins/                # 皮肤资源分包
```

配置步骤：

1. 在 `assets` 下创建用于分包的目录，例如 `assets/subpackages/game`。
2. 在 Cocos Creator 资源管理器里选中该目录。
3. 在属性检查器里找到 Bundle 或 Asset Bundle 配置。
4. 勾选配置为 Bundle。
5. 设置 Bundle 名称，例如 `game`、`rank`、`skins`。
6. 构建微信小游戏或抖音小游戏时，Cocos 会根据 Bundle 配置生成对应分包。

## 5. 代码中如何加载分包

使用分包资源前，需要先加载对应 Bundle。

Cocos Creator 2.x 常见写法：

```ts
cc.assetManager.loadBundle("game", (err, bundle) => {
    if (err) {
        cc.error("加载 game 分包失败", err);
        return;
    }

    bundle.load("prefabs/GamePanel", cc.Prefab, (err, prefab) => {
        if (err) {
            cc.error("加载 GamePanel 失败", err);
            return;
        }

        const node = cc.instantiate(prefab);
        cc.director.getScene().addChild(node);
    });
});
```

Cocos Creator 3.x 常见写法：

```ts
import { assetManager, instantiate, Prefab } from "cc";

assetManager.loadBundle("game", (err, bundle) => {
    if (err) {
        console.error("加载 game 分包失败", err);
        return;
    }

    bundle.load("prefabs/GamePanel", Prefab, (err, prefab) => {
        if (err) {
            console.error("加载 GamePanel 失败", err);
            return;
        }

        const node = instantiate(prefab);
        this.node.addChild(node);
    });
});
```

资源路径通常是 Bundle 内部的相对路径，不需要写文件后缀。

例如资源实际位于：

```text
assets/subpackages/game/prefabs/GamePanel.prefab
```

Bundle 名称是 `game`，加载路径一般写：

```ts
bundle.load("prefabs/GamePanel", cc.Prefab, callback);
```

## 6. 本项目的拆分建议

结合当前项目结构，可以先按功能拆分：

| 模块 | 建议位置 | 原因 |
| --- | --- | --- |
| 登录、首页、基础 UI | 主包 | 首次进入必须显示 |
| `LoginPanel`、`MainPage`、UI 管理器 | 主包 | 启动流程依赖 |
| `GamePanel`、关卡资源、战斗音效、战斗特效 | `game` 分包 | 进入游戏玩法时再加载 |
| `RankPanel`、排行榜展示资源 | `rank` 分包 | 用户点击排行榜时再加载 |
| 皮肤、角色、装饰、大图集 | `skins` 分包 | 非首屏资源，可延迟加载 |
| 微信/抖音开放数据域代码 | 按平台规则单独处理 | 开放数据域有独立运行环境 |

建议先拆最重、最晚使用的资源，不要一开始把所有内容都拆散。

## 7. 常见注意事项

### 7.1 主包不要直接强依赖分包资源

如果主包中的场景、Prefab 或脚本直接引用了分包里的资源，Cocos 可能会为了保证依赖完整，把这些资源重新打进主包，导致分包没有真正减小首包体积。

更稳妥的做法是：

- 主包只保存 Bundle 名称和资源路径。
- 进入对应功能时再调用 `loadBundle`。
- 加载完成后再创建分包里的 Prefab 或使用分包资源。

### 7.2 先加载再使用

分包资源不能假设已经存在。进入战斗、排行榜、皮肤界面前，应先显示加载状态，再加载对应 Bundle 和资源。

### 7.3 控制主包大小

平台会限制主包大小。具体限制可能随平台政策变化，应以微信开发者工具、抖音开发者工具和官方文档提示为准。

### 7.4 不要过度分包

分包太多会增加管理成本，也可能造成频繁下载。通常按功能模块拆分即可，例如：

- `game`
- `rank`
- `skins`
- `activity`

### 7.5 开放数据域不是普通分包

微信小游戏排行榜常涉及开放数据域。开放数据域运行在独立上下文中，目录和通信方式与普通游戏分包不同，不要简单等同于 Asset Bundle 分包。

## 8. 推荐落地流程

1. 先构建一次小游戏，查看主包体积。
2. 找出体积最大的非首屏资源，例如战斗、皮肤、音频、特效。
3. 创建 `assets/subpackages` 目录，并按功能建立子目录。
4. 把非首屏资源移动到对应子目录。
5. 在 Cocos Creator 中把子目录配置为 Asset Bundle。
6. 修改代码，使用 `assetManager.loadBundle` 加载分包资源。
7. 在真机或开发者工具中测试首次进入、进入玩法、打开排行榜等流程。
8. 再次查看构建结果，确认主包体积确实下降。

## 9. 简短结论

小游戏分包不需要放到自己的服务器。它会随小游戏版本上传到平台，由平台负责分发。

项目中只需要在 Cocos Creator 里配置 Asset Bundle，并在代码中按需加载对应 Bundle。
